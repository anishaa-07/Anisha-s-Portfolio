## 🌱 The Story Behind This

I'm a B.Tech student who started this year barely able to explain what a REST API actually was. Six months later I'm running JWT auth, Socket.io real-time chat, and MongoDB schemas in a project I designed myself. I got tired of seeing polished template portfolios that say nothing about *how* someone actually thinks or builds — so I built one that shows the real trail.

```
Todo App  →  DevConnect  →  Huddle
(CRUD          (MERN social      (Real-time collab platform —
 basics)        network)          my first self-designed system,
                                   schema up)
```

I didn't skip steps to look impressive. I built the boring foundational stuff first *because* I wanted the harder project to actually work.

## 🚀 Featured Build — Huddle

> A mini GitHub + Notion + Discord hybrid, built entirely from a schema I designed myself. This is the project I actually think about while falling asleep.

**Why I built it:** every team tool I used for college projects felt scattered — tasks in one app, chat in another, files somewhere else. I wanted to understand *why* that's hard to unify, so I designed one myself instead of just reading about system design.

| Layer | What I used | Why |
|---|---|---|
| Auth | JWT | Understand stateless auth from first principles |
| Real-time | Socket.io | Per-project chat channels, live updates |
| Data | MongoDB + Mongoose | Six entities: User, Team, TeamMember, Project, Task, Message, FileAttachment |
| Files | Cloudinary | Task attachments without managing storage myself |
| Frontend | React + Tailwind | Kanban board, progress dashboard |

**Status:** 🟢 actively in progress — auth and schema done, real-time chat next.
